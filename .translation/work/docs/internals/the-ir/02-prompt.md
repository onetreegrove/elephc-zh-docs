# Translation Instructions

- Target language: Simplified Chinese (`zh-CN`) for Chinese-speaking developers reading `elephc 中文文档`.
- Style: technical, concise, documentation-oriented. Prefer natural Chinese phrasing while preserving the source meaning and specification precision.
- Preserve Markdown structure, heading levels, tables, lists, links, emphasis, and all code fences exactly.
- Do not translate code blocks, inline identifiers, PHP/Rust symbols, opcodes, enum/struct/trait names, file paths, URLs, CLI flags, API names, or runtime/helper names.
- Keep `elephc`, `EIR`, `FFI`, `extern`, `buffer<T>`, `packed class`, `Fiber`, `Mixed`, `AST`, `CLI`, `codegen`, and module/path names unchanged unless the glossary explicitly provides a Chinese explanation.
- Use glossary terms consistently: `compiler` -> `编译器`, `runtime` -> `运行时`, `target` -> `目标平台`, `source map` -> `源码映射`, `static subset` -> `静态子集`, `type checker` -> `类型检查器`, `pointer` -> `指针`, `assembly` -> `汇编`.
- Translate frontmatter human-facing values only; keep YAML keys and `sidebar` order unchanged.
