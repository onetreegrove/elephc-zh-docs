# Translation Instructions

Target: Simplified Chinese (`zh-CN`) for Chinese-speaking developers reading `elephc 中文文档`.

- Use a concise technical documentation style.
- Preserve Markdown structure exactly.
- Preserve frontmatter keys and `sidebar` order; translate only human-facing string values.
- Do not translate code blocks, inline identifiers, PHP symbols, PHP API names, type names, file paths, URLs, CLI flags, or link targets.
- Keep `elephc`, PHP function names, signatures, parameter names, and return types unchanged.
- Apply glossary terms from `.translation/glossary.md`; in this page, use “内置函数” for `Builtins`.
- For tables, translate only natural-language headings such as `Function`, `Signature`, and `Returns`.
