## 算术

| 运算符 | 示例 | 说明 |
|---|---|---|
| `+` | `$a + $b` | 数值加法，或者当两个操作数均为数组时，进行 PHP 数组联合。整型溢出将提升为 `double`。 |
| `-` | `$a - $b` | 减法。整型溢出将提升为 `double`。 |
| `*` | `$a * $b` | 乘法。整型溢出将提升为 `double`。 |
| `/` | `$a / $b` | 除法（总是返回 float） |
| `%` | `$a % $b` | 取模 |
| `**` | `$a ** $b` | 幂运算（右结合，返回 float） |
| `-$x` | `-$x` | 一元取反 |

## 比较

| 运算符 | 示例 | 说明 |
|---|---|---|
| `==` | `$a == $b` | 宽松等值比较，对 bool、null、数值 `int`/`float` 比较、数值字符串以及非数值字符串，使用 PHP 风格的标量类型强制转换 |
| `!=` | `$a != $b` | 宽松不等值比较，使用与 `==` 相同的标量类型强制转换 |
| `===` | `$a === $b` | 严格等值比较（类型和值皆相同） |
| `!==` | `$a !== $b` | 严格不等值比较 |
| `<` | `$a < $b` | 小于 |
| `>` | `$a > $b` | 大于 |
| `<=` | `$a <= $b` | 小于或等于 |
| `>=` | `$a >= $b` | 大于或等于 |
| `<=>` | `$a <=> $b` | 太空船运算符：返回 -1、0 或 1 |
| `instanceof` | `$obj instanceof User` | 运行时类/接口检查；返回 bool |

`instanceof` 支持命名类/接口目标，以及 `self`、`parent` 和 `static`。它还支持动态目标，例如 `$obj instanceof $className`、`$obj instanceof $otherObject`，以及带括号的目标表达式，例如 `$obj instanceof ($prefix . $suffix)`。

直接对象值以及装箱的 `mixed` / 可空 / 联合类型值将在运行时进行检查；在动态目标验证通过后，标量、数组和 null 载荷将返回 `false`。动态字符串目标将与类/接口名称进行不区分大小写的匹配；未知的类字符串将返回 `false`。动态对象目标使用目标对象的运行时类。如果动态目标既不是字符串也不是对象，程序将退出并输出致命的运行时诊断信息。

## 逻辑

| 运算符 | 示例 | 说明 |
|---|---|---|
| `&&` | `$a && $b` | 带短路的逻辑与；优先级高于 `and` |
| `\|\|` | `$a \|\| $b` | 带短路的逻辑或；优先级高于 `or` |
| `and` | `$a and $b` | 单词形式的逻辑与（带短路）；优先级低于 `?:` 和 `??` |
| `or` | `$a or $b` | 单词形式的逻辑或（带短路）；优先级低于 `xor` 和 `and` |
| `xor` | `$a xor $b` | 单词形式的逻辑异或；计算两个操作数 |
| `!` | `!$a` | 逻辑非 |

单词形式的逻辑运算符优先级与 PHP 一致：`and` 的优先级高于 `xor`，而 `xor` 的优先级高于 `or`。这三者的优先级均低于 `&&`、`||`、`??` 以及三元运算符。

单词形式的逻辑运算符不区分大小写（接受 `AND`、`Or` 和 `xOr`）。赋值表达式的优先级高于 `and`、`xor` 和 `or`，这与 PHP 一致：`$x = true and false` 会被解析为 `($x = true) and false`。

## 错误控制

PHP 的错误控制运算符 `@` 用于抑制单个表达式的运行时警告。

```php
<?php
@file_get_contents("missing.txt");
$value = @file_get_contents("missing.txt");
echo "still running";
```

操作数会正常进行求值，并且其值会被保留。只有可抑制的运行时警告会被隐藏；编译时错误和致命运行时错误仍会正常报告。嵌套的 `@` 作用域通过运行时抑制深度进行追踪，且异常展开在进入 `catch` 块之前会恢复之前的抑制状态。

## Print 表达式

`print` 作为 PHP 兼容的表达式受支持：它将其操作数写入到 stdout 并返回 `1`。

```php
<?php
$status = print "ready\n";
echo $status;          // 1
echo print "nested";   // nested1
```

其优先级与 PHP 一致：操作数可以包含 `?:` 和 `??`，而单词形式的逻辑运算符 `and`、`xor` 和 `or` 的优先级低于整个 `print` 表达式。

## 赋值

| 运算符 | 示例 | 等价于 |
|---|---|---|
| `=` | `$x = 5` | 简单赋值 |
| `+=` | `$x += 5` | `$x = $x + 5` |
| `-=` | `$x -= 5` | `$x = $x - 5` |
| `*=` | `$x *= 5` | `$x = $x * 5` |
| `**=` | `$x **= 5` | `$x = $x ** 5` |
| `/=` | `$x /= 5` | `$x = $x / 5` |
| `%=` | `$x %= 5` | `$x = $x % 5` |
| `.=` | `$s .= "x"` | `$s = $s . "x"` |
| `&=` | `$x &= 5` | `$x = $x & 5` |
| `|=` | `$x |= 5` | `$x = $x | 5` |
| `^=` | `$x ^= 5` | `$x = $x ^ 5` |
| `<<=` | `$x <<= 2` | `$x = $x << 2` |
| `>>=` | `$x >>= 2` | `$x = $x >> 2` |
| `??=` | `$x ??= "default"` | 仅当 `$x` 为 `null` 时才赋予右侧的值 |

复合赋值支持本地变量赋值、`for` 循环初始化/更新子句、对象属性、静态属性、索引数组元素、存储在对象属性中的索引数组元素，以及存储在静态属性中的索引数组元素：

```php
$items[0] += 3;
$box->count *= 2;
$box->items[1] >>= 1;
Counter::$count **= 2;
Registry::$items[0] ??= 10;
```

追加目标（如 `$items[] += 1`）是无效的；追加语法仅在简单赋值（`$items[] = 1`）中受支持。
对于非本地复合目标，接收器和索引表达式仅会被求值一次，这与 PHP 在形如 `$items[f()] += 1` 和 `getBox()->count += 1` 等形式下的“读-改-写”行为相匹配。

本地变量赋值也可以用作表达式。该表达式返回被赋予的值，并且依然会更新该本地变量：

```php
<?php
echo ($x = 5);      // 5
echo $x;            // 5

$x = true and false;
echo $x ? "T" : "F"; // T

$count = 4;
echo ($count += 3); // 7
```

赋值表达式的优先级与 PHP 一致：赋值的优先级低于 `?:` 和 `??`，关系弱于单词形式的逻辑运算符。该表达式形式支持本地变量，以及经过稳定化处理的非本地目标（例如数组元素、对象属性、静态属性和存储在属性中的索引数组元素）：

```php
<?php
$items = [1, 2];
echo ($items[1] = 5); // 5

class Box {
    public $count = 1;
    public $items = [2];
}
$box = new Box();
echo ($box->count += 4);
echo ($box->items[0] *= 3);

class Registry {
    public static ?int $value = null;
}
echo (Registry::$value ??= 10);
```

对于非本地赋值表达式目标，在需要时会稳定其接收器（receiver）和索引子表达式，因此具有副作用的目标（例如 `$items[idx()] = 1` 和 `make_box()->count += 1`）只会被求值一次。对于 `=` 和复合赋值，`elephc` 还保留了 PHP 对右侧修改的简单索引（如 `$items[$i] = ($i = 1)`）的求值顺序，同时会对计算后的索引（如 `$items[$i + 0] = ($i = 1)`）进行预求值。

对于 `??=` 表达式形式，`elephc` 保留了 PHP 的短路规则以及非本地目标的条件写入顺序。如果当前目标值非 null，则不会计算右侧的值。如果当前目标值为 null，则右侧的值会在计算最终写入目标之前运行，因此形如 `$items[$i] ??= ($i = 1)` 的形式会通过更新后的简单索引进行写入，而计算后的/有副作用的索引部分仍保持稳定。

## 列表解包

```php
<?php
[$a, , $c] = [10, 20, 30];
echo $a;  // 10
echo $c;  // 30

list($left, $right) = [1, 2];
echo $left . $right; // 12
```

解构支持跳过的条目、嵌套模式、关联键、短语法（`[...]`）以及 `list(...)` 语法。解构的目标可以是本地变量，也可以是普通赋值支持的相同可写非本地形式，例如数组槽位、追加目标、对象属性和静态属性。

```php
<?php
[[$a, $b], [$c, $d]] = [[1, 2], [3, 4]];
["id" => $id, "name" => $name] = ["name" => "Ada", "id" => 7];
[$items[0], $items[]] = [5, 6];
```

与 PHP 一样，在同一个解构模式中不能混用有键（keyed）和无键（unkeyed）的条目。

## 空合并

```php
$x = null;
echo $x ?? "default";    // prints "default"
echo $x ?? $y ?? "last"; // chained — right-associative

$x ??= "fallback";       // assigns because $x is null
$x ??= "ignored";        // keeps "fallback"; RHS is not evaluated
```

`??=` 支持已声明的本地/全局/静态变量，以及非追加属性/数组/静态属性赋值目标，可作为独立的赋值语句使用。
对于具体的本地变量类型，后备值（fallback）必须保持相同的静态类型，或者是一个字面量 `null`。
当后备值可能会改变所存储的运行时表示时，请使用可空、联合或 `mixed` 类型的本地变量。

## 空安全访问

```php
<?php
echo $user?->profile?->name ?? "anonymous";
echo $user?->profile?->label() ?? "missing";
echo $user?->profile->address?->city ?? "unknown";
```

`?->` 支持可空的对象属性访问和方法调用。如果接收器为 `null`，`elephc` 将跳过当前后缀链的其余部分，且表达式返回 `null`；被跳过分支上的方法参数、数组索引和可调用参数将不会被计算。混合链（例如 `$user?->profile->address`）与 PHP 的行为一致：当 `$user` 为 `null` 时会跳过 `->address`，但如果 `$user` 非 null 且 `profile` 本身为 `null`，它仍会正常发出警告或触发致命错误。当您想要提供一个后备值时，请使用 `??`。

空安全访问不能用作赋值目标，也不能与首类可调用对象（first-class callable）的创建结合使用，这与 PHP 一致。

## 三元

```php
$max = $a > $b ? $a : $b;
$label = $name ?: "anonymous";
```

短三元运算符 / Elvis 形式 `expr ?: fallback` 在左侧的值为真值时返回该原始左侧值，否则计算并返回后备值。左侧的表达式只会被求值一次。

## 管道 (PHP 8.5)

管道运算符 `|>` 将左侧的值作为唯一的定位参数来调用右侧的可调用对象。`$x |> foo(...)` 等价于 `foo($x)`。左侧会在右侧之前进行求值，链式管道自左向右应用。

```php
$result = "hello world"
    |> strtoupper(...)
    |> strrev(...);
// equivalent to: strrev(strtoupper("hello world"))
```

右侧可以是任何求值结果为可调用对象的表达式：

- 首类可调用对象语法：`foo(...)`、`Class::method(...)`、`$obj->method(...)`
- 闭包字面量：`(fn($v) => $v + 1)`
- 持有可调用对象的变量：`$cb`
- 任何其他返回可调用对象的表达式

该可调用对象必须接受管道传过来的值作为其第一个参数；其余参数必须是可选的或可变参数（variadic）。管道目标不支持引用传递参数。

首类可调用对象的创建接受本地接收器（如 `$obj->method(...)` 和 `$this->method(...)`），以及非本地接收器表达式（如 `(new Greeter())->greet(...)` 或 `(getThing())->method(...)`）。接收器表达式在创建可调用对象时进行求值，然后捕获在生成的包装器中。

```php
$cb = (new Greeter())->greet(...);
$value |> $cb;
```

其结合性为左结合，优先级低于字符串连接符（`.`）、按位位移和加法运算符（`+`、`-`）。比较、`??`、三元运算符、逻辑运算符以及赋值运算符的优先级均低于 `|>`：

```php
echo 5 + 2 |> double(...);          // (5 + 2) |> double(...)
echo "a" . "b" |> wrap(...);        // ("a" . "b") |> wrap(...)
echo "beep" |> strlen(...) == 4;    // (...|> strlen(...)) == 4
echo $id |> get(...) ?? "default";  // (...|> get(...)) ?? "default"
```
