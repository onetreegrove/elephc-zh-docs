elephc 在两个地方进行优化：降级（lowering）之前的 AST，以及降级之后的 EIR。AST 优化器始终开启。以下 EIR 级别的控制选项允许您关闭优化通道以便进行基准测试和诊断，并在代码形状折中之间进行选择。

## 两个优化层

- **AST 优化器** — 表达在语法上的保留 PHP 语义的重写：常量折叠（constant folding）、常量传播（constant propagation）、控制流剪枝与规范化，以及死代码消除（dead-code elimination）。始终开启；不通过命令行参数控制。参见 [优化器](../internals/the-optimizer.md)。
- **EIR 优化通道** — 针对 AST 无法很好表达的值标识（value identity）、基本块（basic blocks）或支配关系（dominance）的转换。由降级后的不动点通道驱动器运行。通过 `--ir-opt` 进行控制。参见 [EIR 设计](../internals/the-ir.md#optimization-passes)。

## EIR 优化通道

在 AST 降级为 EIR 并通过验证后，不动点通道驱动器会在每个函数上运行已注册的转换通道，直到没有通道报告发生改变。整个流水线运行达到**模块级不动点**：每一轮都会运行跨函数[小函数内联器](#small-function-inlining)，然后在每个函数上驱动单函数通道达到收敛，不断重复直到内联器和函数通道都不再改变任何内容。交错运行让这两个层可以互相促进 —— 内联后的函数体暴露了新的常量和死代码，而简化后的函数则暴露了新的（更小的）内联候选对象。这些通道**默认开启**。

```bash
# Default: EIR optimization passes enabled
elephc hot.php

# Disable them (for A/B comparison or diagnostics)
elephc --no-ir-opt hot.php
elephc --ir-opt=off hot.php

# Explicit enable
elephc --ir-opt=on hot.php
```

环境变量 `ELEPHC_IR_OPT=off` 可以为整个运行禁用这些通道，而无需修改每个命令。

在 debug 和 test 构建版本中，驱动器会在**每次**通道运行后重新验证每个函数，如果通道产生了畸形的 IR 则会中止运行，从而使优化 bug 在开发期间能立即暴露。在 release 构建版本中，这些检查会被编译掉。

### 恒等算术折叠

第一个注册的通道会折叠整数和浮点算术以及按位运算上的代数恒等式：

| 模式 | 结果 |
|---|---|
| `x + 0`, `0 + x`, `x - 0` | `x` |
| `x * 1`, `1 * x`, `x / 1` | `x` |
| `x \| 0`, `x ^ 0`, `x << 0`, `x >> 0` | `x` |
| `x & x`, `x \| x` | `x` |
| `x * 1.0`, `x / 1.0` | `x` |
| `x ^ x`, `x - x`, `x * 0`, `x & 0`, `x % 1` | `0` |

仅应用与 PHP 语义等价的重写：整数 x / 0 和 x % 0 被保留以在运行时触发异常，而浮点加法零（x + 0.0）和 x * 0.0 被排除在外，因为有符号零和 NaN 会使它们变得可观察。

您可以直接使用 [`--emit-ir`](output-and-diagnostics.md#--emit-ir) 查看其效果：

```bash
# With passes on, `$argc * 1` folds away; with --no-ir-opt it stays an `imul`.
elephc --emit-ir app.php
elephc --emit-ir --no-ir-opt app.php
```

这属于窥孔（peephole）级别的优化。它可以加速在热路径中包含冗余恒等运算的代码，对于不包含此类运算的代码则没有任何操作（no-op） —— 这与寄存器分配不同，后者具有更广泛的提升作用。

### 窥孔模式

第二个注册的通道会应用局部重写来清理降级后 EIR 的形状。每个重写都是引用计数平衡（refcount-balanced）的，并且产生与 PHP 完全相同的内容：

| 模式 | 重写 |
|---|---|
| Box/unbox 消除 | 对于标量（`NonHeap`）负载，`unbox(box(x))` → `x` |
| 冗余 `move`/`borrow` | 如果转发操作的结果与操作数具有相同的所有权和类型，则折叠为该操作数 |
| Load/store 转发 | 在对标量局部变量进行 `store` 之后立即对其进行 `load`，将直接读取存储的值 |
| 死存储（Dead store） | 将已持有的值存储回标量局部变量的操作将被移除 |
| Acquire/release 消除 | 如果 `acquire` 的结果仅被其对应的 `release` 所消耗，则将两者全部丢弃 |
| 字符串字面量拼接折叠 | `concat("a", "b")` → 单个字面量 `"ab"` |

Load/store 转发和死存储消除仅适用于未起别名（non-aliased）的标量局部变量，因此绝对不会影响引用语义和写时复制（copy-on-write）。其余模式仅在保留所有权和类型的情况下进行折叠，从而确保清理和引用计数保持平衡。

您可以通过 [`--emit-ir`](output-and-diagnostics.md#--emit-ir) 查看效果：`$x = $argc; echo $x;` 会转发 load 操作，从而使 `echo` 直接读取存储的值，而 `load_local` 则变成了 `nop`。

### 常量折叠

第三个注册的通道会就地将操作数均为编译时常量的操作折叠为单个常量。它涵盖了整数算术（`iadd`、`isub`、`imul`）、按位运算、范围内的移位、单目 `ineg`/`ibit_not`、浮点 `fadd`/`fsub`/`fmul`/`fneg`、有符号整数比较（`icmp`）以及 `is_null`/`is_truthy` 断言。EIR 中的常量值在每次使用时都是相同的常量，因此只需一次前向扫描即可发现常量操作数，并一举折叠类似于 `(2 + 3) * 4` 的链式操作。

每次折叠都完全重现了该操作降级后在运行时计算的结果，因此编译结果绝不会改变：整数在 64 位时回绕（与原生 `add`/`sub`/`mul` 一致），移位仅在计数为 `0..=63` 时进行折叠，而可能会触发异常的整数除法/求模以及对 `NaN` 敏感的浮点除法则保持原样。

配合窥孔优化的标量 load/store 转发（将分配给局部变量的常量移动到其后续使用中），这实现了在 EIR 值 ID *以及*局部槽（local slots）上的单块常量传播：窥孔优化通过局部槽传递常量，而本通道则折叠基于该常量构建的操作数。恒等算术暴露出来的常量也会为此提供输入：`($argc * 0 + 5) * ($argc * 0 + 5)` 简化为 `$argc * 0` → `const_i64 0`（恒等），然后 `0 + 5` → `5`，接着 `5 * 5` → `const_i64 25`（本通道），从而消除了三个 `imul`。

### 公共子表达式消除

第四个注册的通道会在通往某项纯计算的所有路径上均已存在相同计算时，将其移除，并将对其的使用重定向到更早的值。它通过一次支配树值编号（dominator-tree value-numbering）遍历，同时进行块内和跨块消除：一个作用域表将每个纯指令的 `(op, result type, immediate, operands)` 映射到首次计算该指令的值，并且由于是按支配树顺序访问基本块，该表准确地持有了支配当前基本块的定义。因此，匹配项便是一个更早的、处于支配地位的值，重定向是安全的；冗余指令则变成了 `nop`。

只有拥有至少一个操作数且结果为 `NonHeap` 或 `Persistent` 的纯指令（无副作用）才符合条件 —— 它们的值仅取决于其操作数，并且不携带所有堆（owned-heap）的清理工作，因此重写是引用计数中性的。SSA 操作数是按值相等的，所以在相同操作数上运行相同的纯操作会计算出相同的结果。裸常量和地址具体化（`const_*`、`data_addr`）*不会*进行去重：它们重新具体化的成本比保持其活跃更低，因此 CSE 仅针对计算进行优化。使用异常处理的函数会被跳过，因为在运行时，向处理程序的 throw 操作可能会绕过终止符图上的支配者。

结合窥孔优化的 load 转发，`($n + 1) * ($n + 1)` 将仅加载一次 `$n` 并计算一次 `$n + 1`。它所中和的冗余指令会留下死操作数，随后死指令消除会将其移除。

### 循环不变代码外提

第五个注册的通道会将操作数在循环中不发生改变的纯计算移出循环体，并放入循环的前导头（preheader）中，使其只运行一次而不是在每次迭代中都运行。它在支配树上构建循环森林，然后为每个循环将不变集扩展到不动点：当指令的每个操作数要么定义在循环外部（其定义支配前导头），要么其自身正在被提升时，该指令就是不变的。

只有具有至少一个操作数且结果为 `NonHeap`/`Persistent` 的纯指令才会被提升 —— 纯粹性意味着结果仅取决于操作数，且该操作既不读取可变状态也不会触发故障，因此在前导头中对其求值一次（无条件求值，即使其原始基本块仅在某些迭代中运行）是安全的。循环按从内到外的顺序进行处理，因此在多个嵌套循环中保持不变的值会一直移动到最外层的前导头。没有检测到前导头的循环以及使用异常处理的函数会被跳过。

因为 PHP 循环变量存在于局部槽中，并且在每次迭代中都通过非纯的 `load_local` 重新加载，所以一个不变的*源*表达式还不是一个本通道可以提升的纯操作数计算；随着更多值作为 SSA 跨循环流动，其作用范围将会扩大。

### 死指令消除

第六个注册的通道计算 CFG 活跃性，并中和那些未被使用、但其副作用元数据表明其为纯计算的生成结果指令。这清理了早期 EIR 重写所暴露的死值。例如，恒等折叠可以将 `$argc + 0` 转换为 `$argc`；死指令消除随后会从优化后的 EIR 中丢弃现在未被使用的 `const_i64 0`。

该通道被刻意设计得较为保守。即使结果未被使用，它也会保留只读、分配、修改、引用计数、输出、警告、致命错误、throw 以及具备去优化能力的指令。其目标是在不改变 PHP 可见行为或所有权清理的情况下，移除死掉的纯算术和字面量支架。

您可以直接对比形状：

```bash
elephc --emit-ir app.php
elephc --emit-ir --no-ir-opt app.php
```

### 死存储消除

第七个注册的通道会移除在局部槽被覆写或函数退出前其值从未被读取的 `store_local` 写入操作。它在局部槽上计算反向且感知 CFG 的活跃性（`load_local`使槽变活跃，`store_local`将其杀死），因此即使覆写操作在后面的基本块中，死存储也会被丢弃。与窥孔优化的块内、值相等存储丢弃不同，该通道基于活跃性并跨越基本块边界，因此它会移除一个结果从未被观察到的*不同*值的存储。

它仅限于通过普通的 `load_local`/`store_local` 访问且从未通过引用逃逸的非引用计数标量局部变量。引用计数的槽由独立的 `acquire`/`release` 所有权操作包裹，且通过引用传递的调用参数或闭包捕获会通过其 `load_local` 给槽起别名，因此这两者都保持原样，以保持引用计数和别名语义的完整性。

### 分支简化

第八个注册的通道通过以下三种方式剪枝控制流图：

- **常量条件折叠** — 条件为常量（`const_bool`、非零 `const_i64` 或 `const_null`）的 `cond_br` 会变成指向所选分支的无条件 `br`；对常量审查对象进行 `switch` 会折叠到其匹配的 case。例如，`while (true)` 循环降级为常量 `cond_br`，该折叠会将其折叠消除。
- **空块跳转穿透（jump threading）** — 空的、无参数的转发块（仅向前分支的块）的前驱块将被重定向到转发链的末端。
- **不可达块移除** — 从入口起不再可达的块将被中和（终结符设置为 `unreachable`，指令设置为 `nop`）。

使用异常处理的函数会被跳过，因为它们的处理程序基本块可以通过终结符图未显示的隐式边到达。

### 小函数内联

作为模块级不动点的一部分，一个跨函数通道直接将对用户小函数的调用内联到其调用者中；单函数通道随后会优化展开后的函数体，并且由于流水线是迭代运行的，只有在完成此类优化后才变得足够小的被调用者将在后面的轮次中被内联。只有当满足以下**所有**条件时，被调用者才会被内联：

- 其函数体最多包含 **24** 个非 `nop` 指令；
- 它是**非递归的**（直接或互递归） —— 调用图循环分析会排除任何可以到达自身的函数，并且每个调用者的 fuel 上限可以防止终止失败；
- 它不包含任何异常处理操作，并且不是生成器或 Fiber 包装器；
- 它暴露出一个**无析构函数（destructor-free）的边界与函数体**：没有按引用传递或可变参数，并且每个参数、返回值以及每个局部槽都具有无析构函数的类型 —— 即标量（`int`/`float`/`bool`/`string`）以及元素本身也无析构函数的数组/联合体。对象、packed classes（紧凑类）、闭包、资源、泛型 `iterable`/`mixed`、buffers 以及引用单元（ref-cell）/静态/全局/捕获局部变量均被排除在外。

最后一条规则是正确性要求，而不仅仅是启发式规则。拼接操作将被调用者的 `return` 替换为跳转，从而绕过了被调用者隐式的尾声清理工作，因此内联器会重新实现该清理工作的单槽决策：参数槽和直接返回的槽（被调用者将它们排除在外 —— 参数是被借用的，返回值的生命周期所有权会转移给调用者）会被移植，以便宿主尾声忽略它们，而普通的引用计数内部局部变量仍由宿主尾声释放。唯一的残留差异在于这些内部局部变量*何时*被释放（推迟到宿主尾声），这恰恰因为其类型无析构函数而是不可观察的 —— 不会运行 `__destruct`，并且数组不携带可观察的标识。因此，可以运行析构函数的类型（对象，或可能持有对象的数组/mixed）以及按引用参数（它们起别名指向调用者的存储空间）均被排除在外，因为值复制拼接无法重现它们的清理时机或别名关系。与所有 EIR 通道一样，内联由 `--ir-opt` 控制，仅改变性能，绝不改变可观察到的行为。

后续版本将用更多的 EIR 通道扩展此驱动器。

## 寄存器分配

EIR 后端默认使用线性扫描寄存器分配器（linear-scan register allocator，Poletto–Sarkar），在跨调用时将热标量值保留在寄存器中，而不是在每次使用时都将其溢出到栈中。

```bash
# Default: linear-scan registers
elephc hot.php

# Fall back to stack-only placement (spill everything)
elephc --regalloc=stack hot.php
```

`ELEPHC_REGALLOC=stack` 会将此回退应用到整个运行中。栈回退主要用于对比和调试；在计算密集型代码上，线性扫描要明显快得多。

## Null 表示

`--null-repr` 用于选择支持 null 的标量槽的存储方式：

| 值 | 含义 |
|---|---|
| `tagged`（默认） | 内联双字 `{payload, tag}` 标量。 |
| `sentinel` | 单字槽中的带内（in-band）`PHP_INT_MAX - 1` 哨兵（sentinel）（遗留选择性退出）。 |

```bash
elephc --null-repr=sentinel legacy.php
```

`ELEPHC_NULL_REPR` 会为整个运行覆盖默认值。大多数程序应该使用默认值；`sentinel` 作为遗留的选择性退出选项存在。参见 [内存模型](../internals/memory-model.md)。

## 已冻结的遗留后端

`--ast-backend` 选择遗留的直接 AST→汇编后端。它已**弃用**、已被冻结（不再提供新的语言或运行时特性）、会发出警告，并计划在 v0.26.0 版本中移除。请仅在过渡期间使用它来与旧后端的行为进行对比。EIR 后端是默认且唯一活跃的实现目标。
